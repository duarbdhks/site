분석 결론

🎯 규완님이 먼저 가져갈 것은 모델 학습보다 Harness입니다

원문의 전체 지도는 Agent = 모델 + 컨텍스트 + 도구입니다. 10개 장은 기초, 컨텍스트, 기억·RAG, 도구·MCP, Coding Agent, 평가, 후학습, 자기 진화, 멀티모달, 멀티 Agent로 이어집니다. 이 가운데 규완님의 현재 역량과 향후 리드 역할에 가장 직접적인 영역은 Harness 설계, 컨텍스트 공급, 도구와 권한, 실행 검증, 운영 아키텍처입니다.

모델 SFT·RL이나 로보틱스를 제외한 이유는 가치가 없어서가 아닙니다. 현재 단계에서 같은 시간을 투자했을 때 Node.js 백엔드 경험을 즉시 활용하고, 사내 도입과 포트폴리오로 증명할 수 있는 영역의 우선순위가 더 높기 때문입니다. 후학습은 반복 평가 데이터와 고정할 행동이 충분해질 때, 멀티모달은 제품 요구가 생길 때 들어가는 편이 효율적입니다.

2026 Trend Radar

📡 앞으로 2~3년간 남을 가능성이 높은 역량

🛡️ Harness와 격리

모델이 강해질수록 더 넓은 도구 권한을 요구합니다. 샌드박스, 최소 권한, 승인, 재개, 멱등성은 모델 교체와 무관하게 남습니다.

📏 평가 중심 개발

프롬프트를 감으로 조정하는 팀과 실제 업무 평가 세트를 가진 팀의 모델 업그레이드 속도가 달라집니다.

🧠 컨텍스트 효율

도구와 지식이 늘수록 전량 주입보다 점진적 공개, 능동적 발견, 외부 산출물 참조가 중요해집니다.

🔌 프로토콜 상호운용

MCP는 Agent와 도구를, A2A는 서로 다른 Agent를 연결하는 표준 축입니다. 프로토콜보다 권한과 계약 설계가 핵심입니다.

♻️ 외부화 학습

모델 가중치를 바꾸기 전에 성공·실패 경험을 지식, Skill, 코드 도구로 축적해 Agent를 “영리함”에서 “숙련됨”으로 이동시킵니다.

👔 AI-native 팀 운영

Agent가 잘 일하려면 ADR, 실행 가능한 문서, 명확한 완료 정의가 필요합니다. 문서화와 평가가 사람의 협업 구조도 개선합니다.

Course Capstone

🧪 GitHub 이슈 → 검증된 Draft PR Agent

강좌 전체에서 만드는 산출물은 GitHub 이슈를 받아 관련 코드를 탐색하고, 계획 승인을 거쳐 수정·테스트한 뒤 Draft PR을 준비하는 Node.js·TypeScript Agent입니다. 실제 병합과 배포는 범위에서 제외해 안전하게 시작합니다.

GitHub Issue ↓ Context Builder ── 저장소 지도 · ADR · 정책 · Run State ↓ Planner [read-only] ↓ Human Plan Approval ↓ Implementer [sandbox workspace] ├─ glob / grep / read / edit ├─ typecheck / lint / test └─ checkpoint / artifact ↓ Executable Reviewer [fresh test evidence] ↓ Policy Gate + Secret Scan + Diff Scope ↓ Draft PR Publisher ↓ Trace · Eval · Cost Report

제품 목표

반복 이슈의 조사·초기 패치 시간을 줄이되 사람의 최종 코드 리뷰를 유지합니다.

안전 목표

승인 전 쓰기 금지, 테스트 실패 시 PR 금지, 워크스페이스 밖 접근 금지를 코드로 강제합니다.

학습 목표

모델 SDK보다 상태·권한·평가·복구를 중심으로 설명할 수 있는 포트폴리오를 만듭니다.

학습 원칙
코드를 먼저 길게 작성하지 말고 매 세션의 판단 문서와 평가 조건을 함께 남기세요. 같은 모델로 성공률을 올렸다는 증거가 Harness 역량을 가장 잘 보여줍니다.

🧭 이 강좌를 읽는 순서

  1. 처음에는 1→8강 순서로 진행해 공통 캡스톤의 맥락을 유지합니다.
  2. 각 페이지 마지막 셀프 체크를 말로 설명한 뒤 “학습 완료” 버튼을 누릅니다.
  3. 코드 예시는 완성 프레임워크가 아니라 설계 경계를 보여주는 골격입니다. 사용하는 SDK 뒤에 Port·Adapter를 두고 구현하세요.
  4. 회사 적용 전에는 6강 평가와 7강 권한·복구를 생략하지 않습니다.
  5. 원문의 세부 실습은 각 페이지 하단 링크에서 필요할 때 확장합니다.
